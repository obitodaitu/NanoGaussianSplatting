ONE DRAW CALL FOR EVERYTHING
=============================

BRIEF
-----
Currently, each GaussianSplatComponent (tile/proxy) runs its own full GPU pipeline:
  CalcViewData → CalcDistances → 16x RadixSort dispatches → DrawSplats

For N tiles visible at once, that is N × ~19 GPU dispatches and N draw calls.

The "One draw call for everything" technique merges all visible proxies into a single
global GPU buffer at the start of each frame, then runs one global sort and one global
draw call across all splats from all proxies.

New execution flow:
  Phase 1 (per-proxy): N × CalcViewData  →  writes into GlobalViewDataBuffer at a
                        per-proxy offset (GlobalBaseOffset)
  Phase 2 (global):    1 × CalcDistances + 16 × RadixSort + 1 × DrawSplats

Dispatch count: 10 tiles → was ~190 dispatches, becomes ~30 dispatches.


WHY IT HELPS THIS PLUGIN
------------------------
1. CORRECT CROSS-TILE ALPHA BLENDING
   Splats from different proxies that are spatially interleaved are currently NOT sorted
   relative to each other. Each proxy sorts only its own splats. A global sort fixes this
   and produces correct back-to-front order across the entire scene.

2. FEWER GPU DISPATCHES / LOWER OVERHEAD
   GPU dispatch setup and synchronization cost accumulates. Going from ~190 to ~30
   dispatches for 10 tiles reduces driver overhead and improves frame pacing.

3. ONE PIPELINE STAGE INSTEAD OF N
   CalcDistances and RadixSort are the heaviest compute stages. Running them once on the
   merged buffer is more cache-friendly and more efficient than running them N times on
   N small independent buffers.

4. CAMERA-STATIC SKIP SCALES BETTER
   When all proxies are static and the camera hasn't moved, the entire Phase 2 can be
   skipped (already implemented per-proxy; now applies to everything at once).

5. NO HLOD NEEDED FOR DRAW CALL MERGING
   Nanite clusters already handle per-tile LOD (root cluster = few splats when far away).
   This global accumulator replaces what HLOD would do for draw call consolidation,
   without needing to bake combined assets.


NEW DATA STRUCTURES
-------------------
struct FProxySliceInfo
{
    FGaussianSplatSceneProxy* Proxy;
    uint32 GlobalBaseOffset;   // byte/element offset into global buffers
    uint32 SplatCount;
    bool   bNeedsUpdate;       // did this proxy's data change this frame?
};

class FGaussianGlobalAccumulator  (owned by GaussianSplatting module / ViewExtension)
{
    FRWBuffer GlobalViewDataBuffer;      // all proxy ViewData concatenated
    FRWBuffer GlobalSortDistanceBuffer;  // sort keys (distances)
    FRWBuffer GlobalSortKeysBuffer;      // sort values (splat indices)
    FRWBuffer GlobalRadixHistogramBuffer;
    FRWBuffer GlobalIndexBuffer;         // final sorted index buffer for DrawSplats

    uint32 TotalSplatCount;              // sum of all proxy SplatCounts this frame
    TArray<FProxySliceInfo> ProxySlices; // per-proxy offset table
    bool bAnythingChanged;               // if false, skip Phase 2

    // Compaction path additions (for Nanite clusters):
    FRWBuffer GlobalSortIndirectArgs;    // one entry total (GPU-driven count)
    FRWBuffer GlobalSortParamsBuffer;    // {Count, NumTiles}
};
