================================================================================
SOURCE FILE BRIEF
================================================================================
--------------------------------------------------------------------------------
  RUNTIME MODULE -- Public Headers
--------------------------------------------------------------------------------

GaussianSplatting.h
  Module entry point. Creates and owns the ViewExtension and GlobalAccumulator.
  Registers shader directories on startup.

GaussianSplatActor.h / .cpp
  A simple UE Actor wrapper that holds a GaussianSplatComponent as its root.
  Lets you drag-and-drop a Gaussian splat directly into a level.

GaussianSplatComponent.h / .cpp
  UActorComponent that references a GaussianSplatAsset and holds per-instance settings (SH order, opacity/scale multipliers, LOD error thresholds, culling).
  Creates the SceneProxy and reacts to asset changes.

GaussianSplatAsset.h / .cpp
  UObject asset storing the compressed splat data as bulk serialized bytes. Handles compression (positions, rotations, scales, SH, colors), Morton-swizzled color texture creation, cluster hierarchy building, and thumbnail support.

GaussianDataTypes.h
  Core CPU-side data structures: FGaussianSplatData (raw per-splat data), FGaussianChunkInfo (quantization bounds), compression format enums, and a large set of utility functions (Morton codes, SH DC conversion, quaternion packing, splat serialization helpers).

GaussianSplatSceneProxy.h / .cpp
  The FPrimitiveSceneProxy implementation. Owns per-instance RHI GPU resources (position/color/SH buffers, cluster visibility bitmaps, compaction buffers).
  FGaussianSplatGPUResources is the struct that holds all RHI buffers and SRV/UAVs.

GaussianSplatRenderData.h
  Shared render data (ref-counted, one instance per asset shared across proxies). Caches CPU splat/cluster data and manages shared GPU buffers so the same asset data is not duplicated in GPU memory for every component instance.

GaussianClusterTypes.h
  Data structures for the Nanite-style LOD cluster hierarchy: FGaussianCluster (bounding sphere, LOD error, parent/child links) and FGaussianClusterHierarchy.

GaussianClusterBuilder.h / .cpp
  Builds the cluster hierarchy from raw splat data. Sorts splats by Morton code, creates leaf clusters, merges them into parent levels, and computes bounding volumes and error metrics for LOD selection.

GaussianSplatRenderer.h / .cpp
  Static dispatcher that issues all compute and draw calls. Covers: CalcViewData, CalcDistances, RadixSort, ClusterCulling, Compaction, PrepareIndirectArgs, and global accumulator variants. Owns the shader parameter setup logic.

GaussianGlobalAccumulator.h / .cpp
  A shared GPU accumulator for batching multiple proxies into a single draw call. Holds pooled sort-key, distance, and indirect-argument buffers that are reused across all visible proxies each frame.

GaussianSplatViewExtension.h / .cpp
  FSceneViewExtension that tracks registered proxies and fires render-thread callbacks to drive the rendering pipeline each frame.

PLYFileReader.h / .cpp
  Reads binary PLY files (including files larger than 2 GB). Detects SH band count, validates header, handles endianness, and applies per-property linearization (quaternion normalization, exp-scale decoding, sigmoid opacity).

GaussianSplatShaders.h / .cpp
  Declares and implements all C++ shader class wrappers (IMPLEMENT_GLOBAL_SHADER macros) for every compute and graphics shader pass.

--------------------------------------------------------------------------------
  RUNTIME MODULE -- Shaders
--------------------------------------------------------------------------------

GaussianSplatting.hlsl
  The main shared HLSL utility library. Contains: quaternion math, 3D covariance projection, Morton encode/decode, spherical harmonics evaluation, splat data loading from GPU buffers, and packed splat format decoding.

GaussianDataTypes.ush
  HLSL mirror of the CPU data structs. Defines GPU-side FGaussianSplatViewData, FGaussianGPUCluster, FGaussianChunkInfo, plus helper functions (half packing, Morton ops, 2D covariance eigendecomposition).

GaussianSplatting.ush
  Packed splat unpacking helpers: octahedral axis-angle to quaternion, log-scale decoding, uint8 color unpacking. Bridges the packed 16-byte format to full floats.

SphericalHarmonics.ush
  Evaluates spherical harmonic coefficients up to degree 3 for view-dependent color. Called from CalcViewData and the vertex shader.

CalcViewData.usf
  Compute shader. Transforms each visible splat from world space to screen space: projects position to clip coords, computes 2D covariance axes and radii. Also handles cluster LOD splat blending. One thread per splat.

CalcDistances.usf
  Compute shader. Extracts the depth value from each splat's clip position and converts it to a sortable uint32 key for back-to-front ordering.

RadixSort.usf
  GPU LSD radix sort (8-bit, 4 passes). Three compute shaders: CountCS (per-tile histograms), PrefixSumCS (digit offsets), ScatterCS (key redistribution). Supports both direct and indirect dispatch.

ClusterCulling.usf
  Compute shader. Frustum-culls clusters by testing bounding spheres and selects LOD levels based on projected screen-space error. Outputs per-cluster visibility and LOD selection bitmaps.

CompactSplats.usf
  Compute shader. Uses atomics to build a tight list of visible splat indices, filtering by cluster visibility and LOD bitmaps. Eliminates culled/inactive splats before the sort.

PrepareIndirectArgs.usf
  Compute shader. Reads visible splat/cluster counts and writes the indirect dispatch and draw argument buffers so subsequent passes need no CPU readback.

GlobalAccumulatorPrefixSum.usf
  Compute shader. Computes a prefix sum over per-proxy visible splat counts to assign non-overlapping slots in the global accumulator buffers.

GaussianSplatRendering.usf
  Vertex + Pixel shader for rasterization. Expands each splat to a screen-space quad, evaluates the 2D Gaussian alpha, blends additively, encodes velocity for TAA/TSR. Supports cluster debug visualization (color by cluster ID).

--------------------------------------------------------------------------------
  EDITOR MODULE
--------------------------------------------------------------------------------

GaussianSplattingEditorModule.h / .cpp
  Editor module startup/shutdown. Registers all editor-specific factories and type actions.

GaussianSplatAssetFactory.h / .cpp
  UFactory for importing PLY files into GaussianSplatAsset objects via the Content Browser import workflow.

GaussianSplatAssetTypeActions.h / .cpp
  Asset context menu actions in the Content Browser: open properties, reimport, toggle Nanite cluster hierarchy on/off.

ActorFactoryGaussianSplat.h / .cpp
  UActorFactory so dragging a GaussianSplatAsset from the Content Browser into a level automatically spawns a GaussianSplatActor.

GaussianSplatThumbnailRenderer.h / .cpp
  Renders a visual thumbnail for GaussianSplatAsset entries in the Content Browser using the asset's color data.


================================================================================
  GAUSSIAN SPLATTING PLUGIN - RENDER PIPELINE
================================================================================
--------------------------------------------------------------------------------
  STAGE 1 — IMPORT  (Editor only, runs once)
--------------------------------------------------------------------------------

User imports a .ply file via the Content Browser.

  PLYFileReader
    - Reads binary PLY header, detects SH band count (degree 0/1/2/3)
    - Streams vertex data (supports files > 2 GB)
    - Applies linearization: exp() on scales, sigmoid() on opacity,
      normalize quaternions

  GaussianSplatAsset
    - Compresses splat data into bulk-serialized bytes:
        Positions     → chunk-relative quantized (16-bit per axis)
        Rotations     → octahedral axis-angle packed into uint8 x4
        Scales        → log-space uint8 per axis
        Colors (DC)   → Morton-swizzled RGBA8 texture for cache efficiency
        SH bands 1-3  → stored separately, loaded on demand by SH order setting
    - Builds Nanite-style cluster hierarchy (GaussianClusterBuilder):
        Sort splats by 3D Morton code (spatial locality)
        Group into leaf clusters (~64 splats each)
        Merge leaves into parent clusters, compute bounding sphere + LOD error
        Each parent stores a down-sampled "LOD splat" representing its children
    - Saved to .uasset — import is complete

Result: a compressed UObject asset holding all splat data + a cluster tree.


--------------------------------------------------------------------------------
  STAGE 2 — LOAD INTO SCENE  (Game thread / editor placement)
--------------------------------------------------------------------------------

  GaussianSplatActor / GaussianSplatComponent
    - Actor is dragged into the level or spawned at runtime
    - Component holds a reference to the GaussianSplatAsset
    - Per-instance settings live here:
        SH order (0–3), opacity multiplier, scale multiplier,
        LOD error threshold, LOD bias, frustum cull toggle

  GaussianSplatSceneProxy  (created on render thread)
    - One proxy per component instance
    - Allocates per-instance RHI GPU buffers:
        PackedSplatBuffer    — 16 bytes/splat (pos + rot + scale + color index)
        SHBuffer             — spherical harmonic coefficients
        ClusterBuffer        — cluster hierarchy nodes
        LODSplatBuffer       — pre-baked LOD splats for parent clusters
        + working buffers for culling, compaction, sorting (allocated per-proxy)
    - Registers itself with GaussianSplatViewExtension

  GaussianSplatRenderData  (shared, one per asset)
    - Ref-counted; if two components reference the same asset they share
      the static GPU buffers (no duplicate uploads)


--------------------------------------------------------------------------------
  STAGE 3 — RENDER HOOK  (Render thread, every frame)
--------------------------------------------------------------------------------

  UE render pipeline hook point: PostOpaqueRender delegate
    - Fires after all opaque geometry is rendered
    - Before post-processing (tone-map, bloom, etc.)
    - This is where the plugin injects its work each frame

  GaussianSplatViewExtension / OnPostOpaqueRender_RenderThread
    - Collects all registered proxies visible this frame
    - Sorts them front-to-back (closer proxies processed first for budget)
    - Checks render budget (gs.MaxRenderBudget):
        If total splat count exceeds budget, proxies beyond the limit are skipped
    - Decides which render path to use (see Stage 4)

  Camera-Static Skip Cache
    - Compares current ViewProjectionMatrix (TAA/TSR jitter removed) to last frame
    - If camera has not moved AND all proxy states are unchanged:
        Skip Phases 1-3, reuse the previously sorted result
        Jump straight to the draw call
    - Major perf win for static viewpoints


--------------------------------------------------------------------------------
  STAGE 4 — GPU PIPELINE  (Render thread, compute + draw)
--------------------------------------------------------------------------------

Two render paths exist depending on whether all visible proxies have Nanite
cluster data. Both paths end with a single batched draw call.

--------------------------------------------------------------------------------
  PATH A — Full Compaction Path  (Nanite clusters enabled on all proxies)
--------------------------------------------------------------------------------

PHASE 0: Per-proxy  —  Cull, Compact, Prepare indirect args
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  ClusterCulling.usf                                                     │
  │    For each cluster (one thread per cluster):                           │
  │      Test bounding sphere against 6 frustum planes → cull if outside   │
  │      If visible, walk up the parent chain:                              │
  │        projectedError = parentError / distToCamera * screenHeight       │
  │        Apply LOD bias (exponential)                                     │
  │        If error < threshold → select this parent as the LOD level       │
  │        Else → use the leaf cluster (full detail)                        │
  │      Write result bit into ClusterVisibilityBitmap                      │
  │      Write LOD cluster reference into LODClusterBuffer                  │
  └─────────────────────────────────────────────────────────────────────────┘
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  CompactSplats.usf                                                      │
  │    For each splat (one thread per splat):                               │
  │      Check if splat's cluster is visible AND at the selected LOD level  │
  │      If yes → atomically append splat index to CompactedSplatIndices[]  │
  │    Writes final visible count to VisibleSplatCountBuffer                │
  │    Result: tight array of only the splats that need rendering           │
  │    (Can reduce work by ~100x at coarse LOD levels)                      │
  └─────────────────────────────────────────────────────────────────────────┘
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  PrepareIndirectArgs.usf                                                │
  │    Reads VisibleSplatCount, writes GPU indirect dispatch arguments      │
  │    So all subsequent passes dispatch exactly the right thread count     │
  │    without any CPU readback stall                                       │
  └─────────────────────────────────────────────────────────────────────────┘

PHASE 1: Global  —  Prefix sum across all proxies
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  GlobalAccumulatorPrefixSum.usf                                         │
  │    Reads each proxy's VisibleSplatCount into a global array             │
  │    Computes exclusive prefix sums:                                      │
  │      GlobalBaseOffset[i] = sum of visible counts from proxies 0..i-1   │
  │    Proxy i writes its CalcViewData output starting at BaseOffset[i]    │
  │    Also writes final global indirect dispatch + draw arguments:         │
  │      GlobalCalcDistIndirectArgs   = { ceil(TotalVisible/256), 1, 1 }   │
  │      GlobalSortIndirectArgs       = { ceil(TotalVisible/1024), 1, 1 }  │
  │      GlobalDrawIndirectArgs       = { 6, TotalVisible, 0, 0, 0 }       │
  └─────────────────────────────────────────────────────────────────────────┘

PHASE 2: Per-proxy  —  Compute screen-space view data (indirect dispatch)
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  CalcViewData.usf                                                       │
  │    For each VISIBLE splat only (uses IndirectDispatchArgs from Phase 0) │
  │      Decode packed splat: position, quaternion, log-scales, color       │
  │      Evaluate spherical harmonics for view-dependent color              │
  │      Project 3D covariance to 2D screen-space covariance matrix        │
  │      Eigendecompose 2D covariance → two axis vectors + radii           │
  │      Compute clip-space position                                        │
  │      Write to GlobalViewDataBuffer at [BaseOffset[proxyIdx] + localIdx]│
  └─────────────────────────────────────────────────────────────────────────┘

PHASE 3: Global single dispatch  —  Sort all visible splats back-to-front
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  CalcDistances.usf  (indirect dispatch)                                 │
  │    For each visible splat:                                              │
  │      Extract depth from clip-space W                                    │
  │      Convert to sortable uint32 (bit-twiddled for correct float order) │
  └─────────────────────────────────────────────────────────────────────────┘
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  RadixSort.usf  (indirect dispatch, 4 passes of 8-bit LSD radix sort)  │
  │    Pass N (for bits [N*8 .. N*8+7]):                                    │
  │      CountCS   — each tile histograms its 256 digit buckets            │
  │      PrefixSumCS — scan histogram to get per-digit start offsets        │
  │      ScatterCS — redistribute keys+values into sorted order            │
  │    After 4 passes: GlobalSortKeysBuffer is sorted back-to-front        │
  └─────────────────────────────────────────────────────────────────────────┘

--------------------------------------------------------------------------------
  PATH B — Non-Compaction Path  (fallback or Nanite disabled)
--------------------------------------------------------------------------------

  Per-proxy: (optional) ClusterCulling + CalcViewData (direct dispatch, all splats)
  Global:    CalcDistances + RadixSort on total splat count
  No compaction step — all splats (including culled clusters) go through sort

  (Same draw call at the end)

--------------------------------------------------------------------------------
  DRAW CALL  —  Both paths merge here
--------------------------------------------------------------------------------

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  GaussianSplatRendering.usf  —  Vertex Shader (MainVS)                 │
  │    Single instanced draw call (6 vertices per splat = 2 triangles)     │
  │    InstanceId → SortKeysBuffer[InstanceId] → sorted splat index        │
  │    VertexId → quad corner offset (-1/-1 to +1/+1)                      │
  │                                                                         │
  │    Quad expansion:                                                      │
  │      Read screen-space axes (Axis1, Axis2) from GlobalViewDataBuffer   │
  │      NDC position = ClipPos.xy/w + corner.x*Axis1 + corner.y*Axis2    │
  │      Size of quad covers the 2D Gaussian footprint exactly             │
  │                                                                         │
  │    Output: NDC position, local quad coords, color, world position      │
  └─────────────────────────────────────────────────────────────────────────┘
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  GaussianSplatRendering.usf  —  Pixel Shader (MainPS)                  │
  │                                                                         │
  │    Gaussian alpha:                                                      │
  │      distSq = dot(LocalPos, LocalPos)                                  │
  │      alpha  = saturate(exp(-4 * distSq) * splat.opacity)               │
  │      Discard if alpha < 1/255                                           │
  │                                                                         │
  │    Pre-multiplied alpha output:                                         │
  │      RT0.rgb = color.rgb * color.a   (SceneColor, additive blend)      │
  │      RT0.a   = color.a                                                  │
  │                                                                         │
  │    Velocity (for TAA/TSR):                                              │
  │      Reproject world position through previous frame's matrices         │
  │      Strip TAA jitter from both current and previous NDC positions      │
  │      RT1 = EncodeVelocityToTexture(currentNDC - prevNDC)               │
  └─────────────────────────────────────────────────────────────────────────┘

  Blend state (RT0 SceneColor):
    Blend: ONE * src + INVERSE_SRC_ALPHA * dst   (pre-multiplied alpha)
    Depth: DepthNearOrEqual, write enabled
           (splats composite over opaque geometry correctly)


--------------------------------------------------------------------------------
  STAGE 5 — POST-PROCESS  (UE handles this)
--------------------------------------------------------------------------------

  The splat result is now in SceneColor + the velocity buffer.
  UE's standard post-process stack takes over:
    - TSR / TAA uses velocity buffer for temporal accumulation
    - Bloom, tone-mapping, lens flare, etc. apply on top
  The splats look like any other rendered geometry at this point.


--------------------------------------------------------------------------------
  PERFORMANCE OPTIMIZATIONS SUMMARY
--------------------------------------------------------------------------------

  Nanite-style LOD Clusters
    Splats are pre-organized into a hierarchy. Each parent level holds a
    down-sampled representation of its children. Fine detail is shown only
    when the cluster is large on screen.

  Screen-Space Error LOD Selection  (ClusterCulling.usf)
    LOD level is chosen so projected error stays below a pixel threshold.
    Resolution- and FOV-independent: automatically adapts to any screen size.

  Frustum Culling  (ClusterCulling.usf)
    Entire clusters are tested against the view frustum. Clusters behind or
    outside the camera are skipped before any per-splat work begins.

  Splat Compaction  (CompactSplats.usf)
    Only splats belonging to visible, selected-LOD clusters are kept.
    Removes the need to process culled splats in later stages.
    At coarse LOD levels this can be a 100x+ reduction in GPU work.

  GPU-Driven Indirect Dispatch
    Visible splat counts stay on the GPU. PrepareIndirectArgs writes exact
    thread group counts into a buffer; all subsequent passes dispatch
    indirectly from that buffer. No CPU stall waiting for counts.

  Global Accumulator + Single Draw Call
    All proxies share one contiguous GPU buffer. A prefix sum assigns each
    proxy its write slot. The result is one draw call for all Gaussian splats
    in the scene regardless of how many components are active.

  GPU Radix Sort
    Back-to-front order is required for correct alpha blending. A 4-pass
    8-bit LSD radix sort runs entirely on GPU with no CPU involvement.

  Camera-Static Skip Cache
    If the camera and all proxy states are unchanged, the entire compute
    pipeline (culling, view data, sort) is skipped and the previous draw is
    reissued. Zero GPU compute overhead on a static camera.

  Shared Render Data
    Multiple components referencing the same asset share one set of static
    GPU buffers. Placing the same capture multiple times costs nothing extra
    in VRAM for the splat data itself.
