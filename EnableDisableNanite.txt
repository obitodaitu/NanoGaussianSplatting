# Gaussian Splatting Nanite Feature - Per-Asset Enable/Disable Implementation Plan

## Overview
Transform the current global LOD/Nanite system into a per-asset toggle system, similar to Unreal Engine's native Nanite implementation. Users can enable/disable Nanite for each Gaussian Splat asset individually via the Content Browser's Asset Actions menu.

## Current State Analysis
- `gs.UseCompaction` - Global console variable, always ON (setting to 0 crashes)
- `gs.UseLODRendering` - Global console variable to enable/disable LOD rendering
- Cluster hierarchy built during PLY import for ALL assets
- No per-asset Nanite control
- Debug view (`gs.ShowClusterBounds`) shows all assets with cluster data

## Target State
- Compaction always enabled internally (remove console variable)
- Remove global `gs.UseLODRendering` command
- Per-asset `bEnableNanite` property stored in UGaussianSplatAsset
- Nanite toggle via Asset Actions submenu in Content Browser
- Cluster hierarchy only built when user enables Nanite
- Debug view only shows Nanite-enabled assets
- Mixed rendering: Nanite and non-Nanite assets coexist

## Enable/Disable Nanite Workflow

### User Enables Nanite (checks checkbox):
1. System reads source PLY file using stored `SourceFilePath`
2. Builds cluster hierarchy from splat data
3. Generates LOD splats and appends to main buffer
4. Sets `bEnableNanite = true`
5. Asset size INCREASES (cluster + LOD data added)
6. User saves asset to persist changes

### User Disables Nanite (unchecks checkbox):
1. Clears cluster hierarchy data
2. Removes appended LOD splats from buffer
3. Sets `bEnableNanite = false`
4. Asset size DECREASES (back to base splat data)
5. User saves asset to persist changes

### User Re-enables Nanite Later:
- Simply check checkbox again → rebuilds from source PLY file
- Fully reversible, can toggle as many times as needed

### Source File Missing:
- If source PLY file not found when enabling Nanite:
  - Show error dialog: "Source file not found. Please locate the PLY file."
  - Optionally allow user to browse for new source file location

## Notes

### Asset Size Impact:
- Without Nanite: Only base splat data (Position, Rotation, Scale, Color, SH)
- With Nanite: Base data + ClusterHierarchy + LOD splats (appended to main buffer)
- Expected size increase with Nanite: ~20-40% depending on LOD settings

### Performance Considerations:
- Building cluster hierarchy is CPU-intensive (few seconds for 1M+ splats)
- Show progress dialog during cluster building
- Consider async building for better UX

### Backward Compatibility:
- NOT REQUIRED - Old assets can be discarded and reimported
- No migration logic needed for existing assets

### Future Considerations:
- Auto-enable Nanite based on splat count threshold?
- Per-instance Nanite override on components?
- Nanite quality presets (cluster size, LOD aggressiveness)?
